What it does

Polyglot two-store model

An embedded Kuzu property graph (23 node labels, 32 relationship types) holds who-relates-to-whom while a ClickHouse analytical plane (117 tables and views) is a bitemporal, source-precedence event store of what actually happened.

Natural-language ask() seam

A single opaque callable clarifies, routes between graph/facts/hybrid, executes across both stores, and renders results as a WebGL graph or a table.

Structural read-only guard

Every query path is enforced read-only — it strips fences, rejects write/DDL/CALL, blocks smuggled statements, and forces a LIMIT.

Typed-empty contract

A declarative tenant-scoped metadata catalog and bidirectional reconciler give every zero-row result a reason, so only a genuine true_none shows as "none".

Analytical IAM platform

The fact plane adds row-level security, a full IAM/IGA layer, UEBA baselines, an ILP knapsack optimizer, an offboarding blast-radius simulator, and access-graph reachability.

Fulcrum is a living, declaratively-bound model of a whole mining enterprise — people and identities, SaaS accounts and cost, OAuth grants, entitlements and segregation-of-duties, mine operations and OT/SCADA telemetry, equipment, geography, country risk, the gold price, workforce and safety, and legal/M&A history.

It is a genuine polyglot system over two stores: an embedded Kuzu property graph (Cypher — 23 node labels, 32 relationship types) for the who-relates-to-whom, and a ClickHouse analytical plane (117 tables and views) as a bitemporal, source-precedence event store. A single natural-language seam sits on top: one ask() callable clarifies, routes, guards read-only, executes across both stores, and renders as a WebGL graph or a table.

Discipline throughout: a deterministic core with the LLM only at the edges, never-fabricate epistemics, and ~1,400 tests. The public build runs entirely on sanitized fixtures — no PII, ever.
